Git Rebase - короче позволяет изменять историю твоих коммитов путем перемещения текущей branch на другой коммит
Git Fetch - короче загружает коммиты, файлы и ссылки из удаленного репозитория в основной
Git workflow - это набор правил, которые тима использует для работы с гит (Например: git flow, github flow, gitlab flow)
Отличия между git rebase и git merge
git merge: тупо объединяет две ветки и при этом создавая новый коммит (merge commit)
git rebase: все то же, но не создает коммит и переписывает историю

Отличия между git pull и git fetch
Git pull автоматизированная команды fetch, загружает ветку из удаленного репозитория и сразу же объединяет ее с текущей веткой
